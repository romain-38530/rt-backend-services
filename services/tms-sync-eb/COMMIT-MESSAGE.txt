feat(tms-sync): Add comprehensive vigilance system for carriers v2.3.0

Implemented a complete vigilance scoring system for carriers synchronized from Dashdoc.
The system automatically calculates a 0-100% score based on 4 key criteria and classifies
carriers into 5 levels (N1-Premium, N1-Reference, Active, N2-Guest, Observation).

## New Features

### Vigilance Service
- Created services/vigilance.service.js
- Automatic score calculation based on:
  * Legal documents (30%): SIRET, VAT, License
  * Performance (40%): On-time rate/quality score
  * Recent activity (20%): Last order date
  * Order volume (10%): Total number of orders
- 5-level classification system
- Detailed checks with impact breakdown
- Global statistics

### API Endpoints
- GET /api/v1/tms/carriers - List carriers with filters
- GET /api/v1/tms/carriers/:id - Carrier details
- GET /api/v1/tms/carriers/:id/vigilance - Vigilance score
- POST /api/v1/tms/carriers/:id/vigilance/update - Update score
- POST /api/v1/tms/carriers/vigilance/update-all - Update all scores
- GET /api/v1/tms/carriers/vigilance/stats - Global statistics

### Scheduled Jobs
- carriersSync (5 min): Sync carriers from Dashdoc with stats
- vigilanceUpdate (1 hour): Recalculate all vigilance scores
- Detailed logging for monitoring

carriers collection now includes:

- vigilanceScore: Indexed field for fast queries
- vigilanceLevel: Indexed field for filtering
- vigilanceUpdatedAt: Last update timestamp

## Scoring System

Score calculation (100 points max):
- Start: 100 points
- Legal docs missing/invalid: -10 each (max -30)
- Performance <95%: -5 to -40 depending on rate
- Activity >1 week: -3 to -20 depending on age
- Volume <50 orders: -2 to -10 depending on count

Levels:
- 95-100%: N1-Premium
- 85-94%: N1-Reference
- 70-84%: Active
- 50-69%: N2-Guest
- 0-49%: Observation
